# Keep detailed harness progress ephemeral

_Superseded by ADR-0012 for CLI-visible Progress artifacts._

Felix will expose redacted harness progress only to the authenticated Owner console while a turn is running, using the existing authenticated dashboard SSE stream for live delivery. The common Progress contract covers every supported harness in the first implementation: each harness maps its native stream into a small Felix-owned common progress vocabulary, with optional harness-specific details preserved at the adapter boundary. Detail varies by harness capability; lack of native detail is represented honestly as coarse lifecycle state, and the Owner console never fabricates progress detail. Fresh retries are new progress attempts, cancellation becomes `cancelled`, and terminal states clear active progress. Progress is observational only: delivery, serialization, or subscriber failures must not pause, fail, or change harness execution. The live surface may show phase, elapsed time, tool identity, and a short sanitized status, but not private reasoning, full prompts, tool arguments, command output, or arbitrary model text. Felix reports phases and elapsed time rather than a completion percentage unless a harness explicitly supplies a trustworthy total-work measure. The live progress state will be transient; completed turn status and the raw harness log remain durable, but intermediate progress events will not be added to session history. This keeps the session record focused on conversation history, avoids unbounded event growth, prevents transient operational states from being treated as durable facts, and gives the dashboard one stable contract across harnesses.
