# Felix 0.1.1 release checklist

The release path favors speed and simplicity.

- [ ] Merge the intended code to `main`.
- [ ] Open **Actions → Release → Run workflow** and enter `0.1.1`, or push tag `v0.1.1`.
- [ ] Wait for the single AMD64 Docker build.
- [ ] Confirm `frdinawan/felix-agent:0.1.1`, `frdinawan/felix-agent:latest`, and the GitHub Release exist.

No candidate, manual acceptance, security scan, SBOM, attestation, evidence,
multi-architecture smoke test, or separate `latest` promotion is required.
