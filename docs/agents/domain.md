# Domain Docs

This is a single-context repository.

## Before exploring

- Read `CONTEXT.md` at the repo root.
- Read relevant decisions in `docs/adr/`.

## Use the glossary

Use the vocabulary defined in `CONTEXT.md` when naming domain concepts in issues, PRDs, refactors, and tests. If a needed concept is missing, sharpen the term through the domain-modeling workflow before inventing a synonym.

## ADR conflicts

If proposed work conflicts with an existing ADR, call out the conflict explicitly rather than silently overriding it.
